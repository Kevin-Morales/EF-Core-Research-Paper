# Entity Framework Core Research paper

The following is the outline for the repearch paper on object relational mappers and Entity Framework Coure in particular.

1. Introduction:
    a. What is an ORM?
    b. Why use an ORM.
    c. ORM's are not specific to any programming language.
    d. ORM's are integrated in various types of applications, but are popular in serber back-ends.
2. Components of an ORM:
    a. Database drivers (providers).
    b. Context classes.
    c. SQL Generators
    d. Programming language features that power ORM functionality.
    e. Additional tools, such as migrations.
3. A survey of ORM's:
    a. Hibernate, written in Java.
    b. SQL Alchemy, written in Python.
    c. ODB, written in C++.
    d. PHP and JavaScript equivalents.
    e. Miscelaneous (Rust, Go, ETC.).
    f. C-sharp Entity Framework Core.
4. About Entity Framework Core.
    a. The components of EF Core.
    b. Mapping classes and other language constructs to database concepts.
5. Applying EF Core:
    a. Introducing the design of a simple database.
    b. Translating the design to C-sharp classes and EF Core concepts.
6. A possible future of object relational mappers
